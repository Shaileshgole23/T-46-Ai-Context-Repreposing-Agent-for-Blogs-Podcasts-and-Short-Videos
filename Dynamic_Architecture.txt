================================================================================
     AI CONTENT REPURPOSER — DYNAMIC AGENTIC ARCHITECTURE (INDUSTRY GRADE)
================================================================================

Framework  : LangGraph (stateful multi-agent orchestration)
LLM        : Mistral 7B via Ollama (local) / OpenAI GPT-4o (cloud option)
Vector DB  : Pinecone (managed, production-grade)
Embeddings : LLM-based (Mistral embed / OpenAI text-embedding-3-small)
Memory     : LangChain + Pinecone (short + long term)
Search     : Tavily Search API (real-time web search)
Scraping   : Playwright (headless browser, handles JS-heavy pages)


════════════════════════════════════════════════════════════════════════════════
                    WHY DYNAMIC vs STATIC ARCHITECTURE
════════════════════════════════════════════════════════════════════════════════

  STATIC (old):
    Input → fixed pipeline → fixed tools → output
    Every input goes through the same 10 steps regardless of content type.
    No decision making. No adaptation.

  DYNAMIC (new):
    Input → Router Agent decides → specialized agents activate →
    tools selected dynamically → LLM orchestrates → output
    Each input type gets a different processing path.
    Agents think, decide, and adapt based on content.

                                 ▼ Raw Text

  │  LAYER 1 — PERCEPTION AGENT                                             │
  │  Framework: LangGraph node with tool-calling                            │
  │  LLM: Mistral (reasoning + decision making)                             │
  │                                                                         │
  │  This is NOT a fixed pipeline. The agent DECIDES what processing        │
  │  is needed based on the content it receives.                            │
  │                                                                         │
  │  Sub-Agent 1: PREPROCESSING DECISION AGENT                              │
  │    LLM analyzes raw text and decides:                                   │
  │    - Does it need HTML cleaning? (if scraped from web)                  │
  │    - Does it need language detection? (multilingual content)            │
  │    - Does it need OCR correction? (if from PDF with bad formatting)     │
  │    - Does it need deduplication? (if scraped from multiple sources)     │
  │    Tools: nlp/text_cleaner.py, langdetect, NLTK                         │
  │                                                                         │
  │  Sub-Agent 2: CONTEXT ANALYSIS AGENT                                    │
  │    LLM reads the cleaned text and extracts:                             │
  │    - Main topic and subtopics                                           │
  │    - Content type (tutorial / news / opinion / story / research)        │
  │    - Target audience (beginner / expert / general / professional)       │
  │    - Sentiment (positive / negative / neutral)                          │
  │    - Key entities (people, companies, products, locations)              │
  │    - Language and writing style                                         │
  │    Tools: Mistral LLM with structured output (JSON mode)                │
  │                                                                         │
  │  Sub-Agent 3: CHUNKING STRATEGY AGENT                                   │
  │    LLM decides the best chunking strategy based on content type:        │
  │    - Short content (<500 words)  → No chunking, process as whole        │
  │    - Medium content (500-2000)   → Semantic chunking by paragraph       │
  │    - Long content (2000+ words)  → Recursive character splitting        │
  │    - Structured content (lists)  → Section-based chunking               │
  │    Tool: LangChain RecursiveCharacterTextSplitter                       │
  │    Parameters decided by agent: chunk_size, chunk_overlap, separators   │
  │                                                                         │
  │  Output: {                                                              │
  │    cleaned_text, chunks[], content_type, topic, audience,               │
  │    language, sentiment, entities, chunking_strategy_used                │
  │  }                                                                      │
  └──────────────────────────────┬──────────────────────────────────────────┘
                                 │

  ┌─────────────────────────────────────────────────────────────────────────┐
  │  LAYER 2 — EMBEDDING AGENT                                              │
  │  Framework: LangGraph node                                              │
  │  Vector DB: Pinecone (managed, serverless)                              │
  │                                                                         │
  │  This agent decides the best embedding approach for the content.        │
  │  NOT a fixed embedding model — agent selects based on use case.         │
  │                                                                         │
  │  Embedding Strategy Agent:                                              │
  │    LLM decides which embedding model to use:                            │
  │    - English content       → text-embedding-3-small (OpenAI, 1536 dim) │
  │    - Multilingual content  → multilingual-e5-large (1024 dim)          │
  │    - Code content          → code-search-ada-002                        │
  │    - Short social content  → all-MiniLM-L6-v2 (384 dim, fast)          │
  │                                                                         │
  │  LLM-based embedding approach:                                          │
  │    Instead of just encoding text, the agent creates enriched            │
  │    embedding inputs:                                                    │
  │    "Topic: AI Healthcare. Type: Tutorial. Audience: Professionals.      │
  │     Content: [chunk text here]"                                         │
  │    → Richer embeddings that capture context, not just words             │
  │                                                                         │
  │  Pinecone storage:                                                      │
  │    Index: content-repurposer-index                                      │

  │    Metadata stored: {result_id, topic, content_type, tone,              │
  │                       audience, language, created_at, preview}          │
  │    Dimension: 1536 (OpenAI) or 384 (MiniLM)                            │
  │    Metric: cosine similarity                                            │
  │                                                                         │
  │  Output: vectors stored in Pinecone + vector_ids[]                      │

  │  LAYER 3 — RETRIEVAL AGENT                                              │
  │  Framework: LangGraph node with tool-calling                            │
  │  Tools: Pinecone vector search + BM25 keyword search (hybrid)           │
  │                                                                         │
  │  This agent performs HYBRID RETRIEVAL — combines vector search          │
  │  with keyword search for better results.                                │
  │                                                                         │
  │  Step 1: Vector Search (semantic)                                       │
  │    Tool: Pinecone query API                                             │
  │    → Finds semantically similar past content                            │
  │    → top_k = 5, filter by content_type if available                    │
  │                                                                         │
  │  Step 2: Keyword Search (lexical)                                       │
  │    Tool: BM25 index (sparse vectors in Pinecone)                        │
  │    → Finds exact keyword matches in past content                        │
  │    → Catches cases where semantic search misses specific terms          │
  │                                                                         │
  │  Step 3: Reranking Agent                                                │
  │    LLM reranks the combined results:                                    │
  │    - Scores each retrieved document for relevance (1-10)               │

  │  Step 4: Context Injection                                              │

════════════════════════════════════════════════════════════════════════════════
                    LANGGRAPH STATE GRAPH STRUCTURE
════════════════════════════════════════════════════════════════════════════════

  Nodes (each is an agent):
    input_router       → classifies input type
    search_scrape      → Tavily + Playwright
    youtube_extract    → yt-dlp + Whisper
    document_extract   → PyMuPDF + python-docx
    perception         → preprocessing + context analysis + chunking
    embedding          → embedding strategy + Pinecone upsert
    retrieval          → hybrid search + reranking
    memory_read        → fetch session + long-term memory
    orchestrator       → central brain, dispatches to generators
    summarize          → generation agent 1
    highlights         → generation agent 2
    script             → generation agent 3
    social_posts       → generation agent 4
    titles             → generation agent 5
    thread             → generation agent 6
    linkedin           → generation agent 7
    keywords_hashtags  → generation agent 8
    reflection         → quality scoring + self-improvement
    memory_write       → update Pinecone + LangChain memory
    storage            → SQLite save

  Edges (conditional routing):
    input_router → [search_scrape | youtube_extract | document_extract | perception]
    perception → embedding → retrieval → memory_read → orchestrator
    orchestrator → [all 8 generation agents in parallel]
    all generators → reflection
    reflection → [storage if score>=7 | regenerate if score<7]
    storage → memory_write → END

  State object passed between all nodes:
    {
      input_type, raw_text, cleaned_text, chunks[],
      content_analysis{topic, type, audience, language},
      embeddings[], retrieved_docs[], memory_context,
      generated_outputs{}, quality_scores{},
      agent_trace[], total_time
    }
